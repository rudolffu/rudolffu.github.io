A brief guide to install ATLAS in Ubuntu

  • Nov 24, 2021
  • 2 minutes read

1. Preparations

1.1 Download ATLAS package

The main ATLAS homepage is at: http://math-atlas.sourceforge.net/. The latest version of ATLAS is 3.10.3. If you want to install ATLAS along with LAPACK, you can download LAPACK from https://www.netlib.org/lapack/.

1.2 Make directories for ATLAS

The following scripts assume you install ATLAS with root privilege. If you do not have root privilege, you can put everything in ~/opt/ instead of /opt/ and change environment variables after building the packages.

mkdir /opt/numerics
mkdir /opt/lib
mkdir /opt/lib/atlas
cp ~/Downloads/atlas3.10.3.tar.bz2 /opt/numerics/
cp ~/Downloads/lapack-3.10.0.tar.gz /opt/numerics/
cd /opt/numerics/
bunzip2 -c atlas3.10.3.tar.bz2 | tar xfm -
mv ATLAS ATLAS3.10.3

1.3 Check Fortran compiler

According to the ATLAS documentation:
For most systems, ATLAS defaults to using the Gnu compiler collection for its ATLAS install. This means configure will automatically search for either g77/gcc or gfortran/gcc. If it can’t find them, it will typically stop with an error message. For some platforms, ATLAS knows good flags to use for multiple compilers, and so you may get good flags by simply changing the compiler name. If this doesn’t work, you’ll need to specify both the compiler name and the flags to use. For the fortran compiler, you can switch the fortran compiler without performance or install penalty on all platforms. To do so, simply add the flags:

-C if <fortran compiler with path> -F if 'fortran flags'

to the configure command. If you need to install ATLAS on a platform that doesn’t have a working fortran compiler, you can do so by adding the flag: –nof77 instead. Note that the fortran interface to BLAS and LAPACK cannot be built without a fortran compiler.

gfortran -v

If an error occurs, you may need to install the Fortran compiler. For example:

sudo apt update
sudo apt install gfortran
